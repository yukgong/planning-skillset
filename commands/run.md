## 🎯 명령어 목적

기획 스킬셋의 8개 에이전트 중 하나를 선택하여 실행합니다.

- 특정 에이전트 실행
- 기획 플로우 진행
- 진행 상황 추적
- 이전 진행상황 계속

## 📋 사용 패턴

### 패턴 1: 에이전트 선택 UI (인수 없음)

```bash
/skillset:run
```

아래와 같은 메뉴를 표시합니다:

```
🎯 기획 스킬셋 - 에이전트 선택

현재 진행 상황:
[0/8] 기획 준비 완료

다음 에이전트를 선택하세요:

① 🔍 리서치 에이전트
   시장 정보 수집 및 분석 기초 제공
   소요시간: 6-8시간 | 난이도: 중

② 📊 분석 에이전트
   수집된 데이터 분석 및 인사이트 도출
   소요시간: 5-7시간 | 난이도: 중
   (리서치 완료 후 활성화)

③ 💡 아이데이션 에이전트
   분석 기반 창의적 아이디어 생성
   소요시간: 6-8시간 | 난이도: 높음
   (분석 완료 후 활성화)

④ 🏗️ 구조화 에이전트
   아이디어를 구체적인 제품 개요로 구조화
   소요시간: 7-9시간 | 난이도: 높음
   (아이데이션 완료 후 활성화)

⑤ ✔️ 검증 에이전트
   제품 계획의 체계적 검증
   소요시간: 4-6시간 | 난이도: 중
   (구조화 완료 후 활성화)

⑥ 👥 사용자중심 에이전트
   사용자 경험(UX) 최적화
   소요시간: 6-8시간 | 난이도: 높음
   (구조화 완료 후 활성화)

⑦ 📢 커뮤니케이션 에이전트
   이해관계자와의 효과적인 소통
   소요시간: 5-7시간 | 난이도: 중
   (검증 완료 후 활성화)

⑧ 🎯 전략 에이전트
   장기 사업 전략 수립
   소요시간: 5-7시간 | 난이도: 높음
   (검증 완료 후 활성화)

선택: (번호 입력)
```

### 패턴 2: 특정 에이전트 직접 실행

```bash
/skillset:run research          # 리서치 에이전트 실행
/skillset:run analysis          # 분석 에이전트 실행
/skillset:run ideation          # 아이데이션 에이전트 실행
/skillset:run structure         # 구조화 에이전트 실행
/skillset:run validation        # 검증 에이전트 실행
/skillset:run user-centered     # 사용자중심 에이전트 실행
/skillset:run communication     # 커뮤니케이션 에이전트 실행
/skillset:run strategy          # 전략 에이전트 실행
```

### 패턴 3: 가이드 포함 실행

```bash
/skillset:run research --guide          # 리서치 에이전트 + 가이드 표시
/skillset:run analysis --guide          # 분석 에이전트 + 가이드 표시
```

가이드를 먼저 표시한 후 에이전트 실행 여부 확인:

```
📖 리서치 에이전트 가이드

[이전 50줄 요약]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

계속 진행하시겠습니까?
① 에이전트 실행 시작
② 전체 가이드 보기
③ 취소
```

### 패턴 4: 이전 진행상황 계속

```bash
/skillset:run --continue
```

마지막으로 실행한 에이전트를 다시 실행합니다:

```
📍 마지막 진행상황: 분석 에이전트 (진행중)

계속하시겠습니까?
① 분석 에이전트 재개
② 다른 에이전트 선택
③ 취소
```

## 🔄 실행 흐름

```
User Input (/skillset:run [args])
    ↓
[Step 1] 인수 파싱
    ├─ agent-name 추출
    ├─ 플래그 확인 (--guide, --continue)
    └─ 옵션 적용
    ↓
[Step 2] 에이전트 유효성 검증
    ├─ 에이전트 ID 확인
    ├─ 선행 조건 확인
    │  ├─ 필요한 이전 에이전트 완료 여부 확인
    │  ├─ 관련 산출물 존재 여부 확인
    │  └─ 조건 불충족 시 사용자에게 알림
    └─ 에이전트 메타데이터 로드
    ↓
[Step 3] 선택 UI (에이전트 미지정 시)
    ├─ AskUserQuestion으로 에이전트 선택
    ├─ Progressive Disclosure 적용
    └─ 선택한 에이전트 ID 획득
    ↓
[Step 4] 가이드 표시 (--guide 옵션)
    ├─ 가이드 파일 로드
    ├─ 처음 50줄 표시
    └─ 계속 진행 여부 확인
    ↓
[Step 5] 에이전트 위임
    ├─ Task() 호출로 에이전트 실행
    ├─ agent_id, context 전달
    └─ 진행 상황 업데이트
    ↓
[Step 6] 완료 처리
    ├─ 산출물 정렬
    ├─ 메타데이터 업데이트
    └─ 다음 권장 단계 제시
    ↓
[Step 7] 다음 단계 제안
    ├─ 완료된 에이전트 표시
    ├─ 활성화된 다음 에이전트 제시
    └─ 권장 동작 제시
```

## 🎯 선행 조건 (Prerequisites) 맵

```
리서치 → (필수 없음)

분석 → 리서치 필수
아이데이션 → 분석 필수
구조화 → 아이데이션 필수

검증 → 구조화 필수
사용자중심 → 구조화 필수
커뮤니케이션 → 검증 필수
전략 → 검증, 사용자중심 권장 (검증 필수)
```

## 💡 팁

- **병렬 실행**: 독립적인 에이전트(예: 사용자중심 + 커뮤니케이션)는 순서 무관하게 실행 가능
- **재실행**: 완료한 에이전트도 언제든 재실행 가능 (산출물 덮어쓰기)
- **진행 상황 추적**: /skillset:list 로 현재 진행 상황 확인
- **가이드 활용**: 에이전트마다 상세 가이드가 있습니다 (--guide 옵션)

---

**명령어 예시:**
```bash
/skillset:run                      # 에이전트 선택 UI
/skillset:run research             # 리서치 에이전트 실행
/skillset:run analysis --guide     # 분석 에이전트 + 가이드
/skillset:run --continue           # 마지막 진행상황 계속
```
