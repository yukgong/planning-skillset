# 검증 에이전트 (Validation Agent) 가이드

## 📋 개요

검증 에이전트는 기획된 아이디어나 전략이 실제로 실행 가능하고 가치있는지를 체계적으로 검증합니다. 시장성, 기술적 타당성, 경제적 실현성, 그리고 리스크를 종합적으로 평가하여 Go/No-Go 의사결정을 지원합니다.

### 🎯 목표
- 기획안의 시장 타당성 검증
- 기술적 실현 가능성 평가
- 경제적 수익성 검증
- 주요 리스크 식별 및 완화전략 수립
- 최종 의사결정 기준 제시

### 💡 언제 사용하는가?
- 새로운 기능이나 제품 기획이 완료되었을 때
- 기획안에 대한 실현 가능성이 불명확할 때
- 투자 의사결정이 필요할 때
- 시장 진출 전 최종 검토가 필요할 때
- 리스크를 사전에 식별하고 대비하고 싶을 때

## 🎯 단계별 프로세스

### 📌 Step 1: 검증 기준 정의 (1-2시간)

**목표:** 검증 대상을 명확히 하고 평가 기준을 수립합니다.

#### 실행 방법

1. **기획안 분석**
   - 검증할 기획의 핵심 가치 제안(Value Proposition) 파악
   - 타겟 시장 및 고객 확인
   - 기획안의 주요 가정(Assumptions) 도출

2. **검증 항목 정의**
   - 시장성 검증 (Market Viability)
   - 기술성 검증 (Technical Feasibility)
   - 경제성 검증 (Economic Viability)
   - 리스크 검증 (Risk Assessment)

3. **평가 기준 수립**
   ```
   시장성: 시장 규모, 경쟁 강도, 진입장벽
   기술성: 현재 기술 수준, 개발 난이도, 보유 역량
   경제성: ROI, 수익성, 수익 창출 기간
   리스크: 시장 리스크, 기술 리스크, 조직 리스크
   ```

4. **검증 우선순위 결정**
   - 고위험 항목 우선 검증
   - 의사결정 시간 고려
   - 리소스 가용성 평가

#### 산출물 예시

```yaml
검증 기준 정의서:
기획명: 협업도구 SaaS - AI 기반 회의 요약 기능

검증 항목별 기준:
1. 시장성
   - 유효 시장규모(TAM): $2B+ 달성 필요
   - 경쟁사 수: 3-5개 이상 (포화도 확인)
   - 진입장벽: 평가 점수 3/5 이상 필요

2. 기술성
   - 핵심 기술 확보: 음성인식, 자연어처리
   - 개발 일정: 6개월 이내
   - 성공 확률: 70% 이상

3. 경제성
   - 초기 투자: $500K 이하
   - ROI: 24개월 이내 달성
   - 3년 누적 수익: $5M+ 예상

4. 리스크
   - Critical 리스크 허용 불가
   - High 리스크 2개 이상 발생 시 중단
   - 완화 가능성: 70% 이상 필요
```

### 📌 Step 2: 시장성 검증 (3-5시간)

**목표:** 시장 수요, 경쟁 상황, 진입 가능성을 검증합니다.

#### 실행 방법

1. **시장 규모 분석**
   - TAM(Total Addressable Market) 계산
     - Bottom-up: 고객 수 × 평균 가격
     - Top-down: 전체 시장 규모 추정
   - SAM(Serviceable Addressable Market) 정의
   - SOM(Serviceable Obtainable Market) 목표 수립

2. **경쟁 환경 분석**
   - Direct 경쟁사 (동일한 기능/시장)
   - Indirect 경쟁사 (대체 솔루션)
   - Potential 경쟁자 (진출 가능성)
   - 경쟁사별 강점/약점 비교

3. **고객 수요 검증**
   - 고객 인터뷰 (10-20명)
   - 설문 조사 (최소 100명)
   - 기존 고객 피드백 (해당시)
   - 시장 트렌드 분석

4. **진입 가능성 평가**
   - 시장 진입 장벽 평가
   - 규제 환경 검토
   - 파트너십 가능성
   - 비용 대비 시장 점유율 달성 가능성

#### 검증 프레임워크

```
시장성 평가 매트릭스:

항목              | 평가 기준      | 점수 | 결론
-----------------|----------------|------|------
시장 규모        | $100M+ 이상   | 9/10 | ✓ 충분
성장률           | CAGR 20%+     | 8/10 | ✓ 양호
경쟁 강도        | 3-5개 경쟁사  | 7/10 | ⚠ 주의
진입 장벽        | 중간 이상     | 7/10 | ⚠ 고려
고객 수요        | 80% 관심도    | 8/10 | ✓ 양호
진입 가능성      | 12개월 내     | 8/10 | ✓ 가능

종합 평가: 77/100 - PASS (시장성 검증 완료)
```

### 📌 Step 3: 기술성 검증 (2-4시간)

**목표:** 기술적 실현 가능성과 개발 난이도를 평가합니다.

#### 실행 방법

1. **핵심 기술 검증**
   - 필요 기술 목록화
   - 현재 보유 역량 평가
   - 부족한 기술 식별
   - 외부 솔루션 검토

2. **아키텍처 설계 타당성**
   - 시스템 아키텍처 검토
   - 확장성/성능 평가
   - 보안 고려사항
   - 통합 가능성

3. **개발 난이도 평가**
   - 기술 스택 평가 (적절성, 팀 역량)
   - 주요 리스크 기술 식별
   - MVP 기능 vs 전체 기능
   - 개발 일정 현실성

4. **프로토타입/PoC 검증**
   - 핵심 기술 검증용 PoC 결과
   - 성능 테스트 결과
   - 통합 테스트 결과
   - 사용자 테스트 피드백

#### 기술성 검증 프레임워크

```
기술성 평가 점수표:

기술 항목        | 평가 결과           | 점수 | 리스크
-----------------|-------------------|------|-------
음성인식 정확도  | 94% 달성 (목표 90%) | 9/10 | 낮음
자연어처리       | 기술 검증 완료      | 8/10 | 낮음
다중언어 지원    | 베타 버전 테스트    | 6/10 | 중간

데이터 보안      | SSL, 암호화 적용    | 9/10 | 낮음

전체 기술성 점수: 78/100 - PASS
리스크: 중간 2개 (완화 계획 필요)
```

### 📌 Step 4: 경제성 검증 (2-3시간)

**목표:** 수익성과 투자 수익률을 검증합니다.

#### 실행 방법

1. **수익 모델 정의**
   - 주요 수익원 (SaaS 구독, 프리미엄 기능, B2B)
   - 가격 책정 전략
   - 고객군별 수익 예상
   - 확대 시나리오

2. **비용 구조 분석**
   - 초기 개발 비용
   - 운영 비용 (서버, 인력, 마케팅)
   - 고객 획득 비용 (CAC)
   - 고객 유지 비용

3. **재무 예측**
   - Break-even point 계산
   - 3년 매출 예상
   - ROI 계산
   - 현금흐름 분석

4. **수익성 민감도 분석**
   - 가격 ±10% 영향도
   - 고객 확보 ±20% 영향도
   - 비용 ±10% 영향도

## ❓ Q&A

### Q1: 검증에 얼마나 많은 시간을 투자해야 하나요?

**A:** 일반적으로 3-4주 소요됩니다.
- Step 1-2: 1주 (기준 정의 + 시장성)
- Step 3-4: 1주 (기술성 + 경제성)
- Step 5-6: 1주 (리스크 + 완화)
- Step 7-8: 3-5일 (의존성 + 최종 보고)

조직 규모에 따라 단축 가능:
- 소규모팀 (5명 이하): 2주 가능
- 중규모팀 (5-15명): 3주 권장
- 대규모조직 (15명 이상): 4주 이상

### Q2: 검증에 실패하면 어떻게 하나요?

**A:** 실패는 기획을 변경하여 재검증하는 것을 의미합니다.
- No-Go 결정: 기획 중단 또는 대폭 수정
- Conditional Go: 조건 충족 시 재검증
- 부분 실패: 해당 영역만 재검증

예: 시장성 PASS인데 기술성 FAIL
→ 기술 구조 단순화 후 기술성 재검증

### Q3: 누가 검증을 주도해야 하나요?

**A:** 일반적으로 CEO 또는 Product Manager가 주도합니다.
- CEO: 전체 검증 프로세스 감독
- Product Manager: 시장/기술성 주도
- CFO: 경제성 분석 주도
- CTO: 기술성 검증 주도
- 전략팀: 리스크 분석 주도

### Q4: 부분적 검증도 가능한가요?

**A:** 가능하지만 권장되지 않습니다.
- 최소한 시장성 + 기술성은 필수
- 경제성만 건너뛰면: 매우 위험 (적자 기획)
- 리스크 평가 없으면: 숨겨진 이슈 누락

권장: 모든 단계 검증 수행

### Q5: 검증 결과가 Go인데 실패한 경우?

**A:** 실패 원인 분석 후 개선합니다.
- 검증 오류 분석: 가정 검토, 데이터 재확인
- 실행 오류 분석: 계획 대비 실행 편차 점검
- 시장 변화 분석: 외부 요인 변화 검토

조직 학습 자료로 활용하여 향후 검증 정확도 향상

### Q6: 검증 중 새로운 정보 발견 시?

**A:** 즉시 검증 기준 또는 결과를 업데이트합니다.
- 새로운 경쟁사 발견: Step 2 재검증
- 핵심 기술 한계 발견: Step 3 재검증
- 시장 축소 신호: Step 2 재평가

검증은 동적 프로세스입니다.

### Q7: Go/No-Go 결정이 어려울 때?

**A:** AskUserQuestion으로 의사결정권자의 입력을 받습니다.
- "이 조건은 반드시 충족해야 하나?" 확인
- "몇 가지 조건은 완화 가능한가?" 협의
- "Conditional Go로 진행할 수 있나?" 검토

의사결정은 데이터 기반이지만 최종은 경영진 판단입니다.

### Q8: 검증 후 ROI는 어떻게 측정하나요?

**A:** 사전 예측과 실제 결과를 비교합니다.
- 예측 ARR vs 실제 ARR
- 예측 Break-even vs 실제 Break-even
- 리스크 발생률 vs 예측 발생률

매분기 검증 정확도를 점검하여 향후 개선에 반영합니다.
