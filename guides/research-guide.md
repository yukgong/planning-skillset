# 🔍 리서치 에이전트 상세 가이드

> 웹 애플리케이션 기획의 첫 번째 단계
> 시장 정보 수집, 경쟁사 분석, 사용자 니즈 파악

---

## 📋 개요

### 에이전트 목표
**시장 정보를 광범위하게 수집하여 이후 기획 단계의 기초를 제공합니다.**

### 언제 사용하는가?

✅ **필수**:
- 새로운 서비스 기획 시 (초기 단계)
- 새로운 시장 진출 검토 시
- 경쟁 상황을 파악해야 할 때
- 사용자 니즈를 정확히 알아야 할 때

❌ **불필요**:
- 내부 기능 개선 (이미 시장 이해도 높을 때) → 분석 에이전트부터 시작
- 긴급 피처 개발 (시간 부족) → 구조화 에이전트부터

### 기대되는 결과

| 산출물 | 설명 | 용도 |
|--------|------|------|
| 리서치 보고서 | 시장 현황, 트렌드, 핵심 발견사항 | 분석 에이전트 입력 |
| 경쟁사 분석표 | 경쟁사 기능, 가격, 전략 비교 | 차별화 포인트 발굴 |
| 사용자 인터뷰 요약 | 사용자 니즈, 통증점, 기대사항 | UX 설계 기초 |
| 시장 트렌드 분석 | 산업 동향, 기술 트렌드 | 전략 수립 기초 |

---

## 🎯 단계별 프로세스 (8 Steps)

### **Step 1: 리서치 목표 및 범위 정의** (1-2시간)

#### 목표
리서치를 시작하기 전에 명확한 목표와 범위를 정의합니다.

#### 실행 방법

**1-1. 리서치 질문 정의**
```
예시:
Q1: "협업 도구 시장에서 AI 기능을 원하는 사용자는 누구인가?"
Q2: "기존 협업 도구(노션, 아사나)의 한계점은 무엇인가?"
Q3: "우리가 차별화할 수 있는 영역은?"
```

**1-2. 조사 대상 결정**
- 지역: 국내? 글로벌?
- 산업: 스타트업? 대기업? 모두?
- 사용자: PM? 팀리더? 개별 워커?

**1-3. 리서치 예산/기간 결정**
- 시간: 1주? 2주? 1개월?
- 리소스: 내부 팀? 외부 리서처?
- 비용: 예산 범위

#### 체크리스트
- [ ] 리서치 질문 3-5개 정의
- [ ] 조사 대상 명확히
- [ ] 예산/기간/리소스 결정
- [ ] 팀 동의 확보

---

### **Step 2: 시장 규모 및 현황 조사** (3-5시간)

#### 목표
시장의 규모, 성장률, 현재 플레이어를 파악합니다.

#### 실행 방법

**2-1. 시장 규모 조사**
```
조사 항목:
- 시장 규모: 2024년 협업 도구 시장 규모는?
- 성장률: 연 몇% 성장 중인가?
- 주요 플레이어: 누가 시장을 주도하는가?

조사 방법:
- Statista, Grand View Research 등 시장 보고서
- 각 회사 공시 자료
- 산업 뉴스 (Techcrunch, VentureBeat 등)
```

**2-2. 시장 세분화**
```
조사 항목:
- 엔터프라이즈 시장 vs 중소기업 시장 vs 개인
- 특정 산업 (IT, 금융, 제조 등) 수요
- 지역별 시장 차이

결과:
협업 도구 시장
├─ 엔터프라이즈: $X 규모, Y% 성장
├─ 중소기업: $X 규모, Y% 성장
└─ 개인: $X 규모, Y% 성장
```

**2-3. 시장 트렌드 파악**
```
최근 트렌드:
- AI 기반 자동화 (업무 자동 스케줄링)
- 리모트 워크 친화적 기능
- 보안/프라이버시 중시
- API 기반 통합 플랫폼화
```

#### 산출물
- 시장 규모 및 성장률 정리
- 주요 플레이어 리스트
- 시장 트렌드 정리 (3-5개 항목)

#### 체크리스트
- [ ] 시장 규모 데이터 수집 (신뢰할 수 있는 출처)
- [ ] 세분화된 시장 분석
- [ ] 현재 트렌드 파악
- [ ] 데이터 출처 기록

---

### **Step 3: 경쟁사 선정** (30분-1시간)

#### 목표
분석할 경쟁사 3-5개를 선정합니다.

#### 실행 방법

**3-1. 경쟁사 식별**
```
직접 경쟁사:
- Notion (올인원 워크스페이스)
- Asana (프로젝트 관리)
- Monday.com (업무 자동화)

간접 경쟁사:
- Slack (커뮤니케이션)
- Google Workspace (생산성 도구)
```

**3-2. 경쟁사 선정 기준**
```
선정 기준:
1. 직접 경쟁: 기능/타겟 사용자 겹침
2. 시장 점유율: 상위 플레이어
3. 혁신성: 새로운 접근법
4. 우리와의 유사성/차이점

선정 수: 3-5개 (너무 많으면 분석 시간 증가)
```

#### 산출물
- 경쟁사 리스트 3-5개 (직접/간접 분류)
- 선정 이유 간단히 기록

#### 체크리스트
- [ ] 직접/간접 경쟁사 구분
- [ ] 최소 3개 이상 선정
- [ ] 선정 이유 기록

---

### **Step 4: 경쟁사 상세 분석** (5-8시간)

#### 목표
선정된 경쟁사의 기능, 가격, 전략을 상세히 분석합니다.

#### 실행 방법

**4-1. 기능 비교 분석**
```
분석 항목:
1. 핵심 기능
   - Notion: 노트, DB, 칸반보드, 타임라인, 캘린더
   - Asana: 칸반, 타임라인, 포트폴리오, 자동화, 리포팅
   - Monday: 칸반, 타임라인, 자동화, AI 기반 인사이트

2. 차별화 기능
   - Notion: 자유도 높은 데이터베이스
   - Asana: 엔터프라이즈급 리포팅
   - Monday: AI 기반 업무 자동화

3. 사용자 경험
   - 인터페이스 복잡도
   - 학습곡선
   - 모바일 지원
```

**4-2. 가격 모델 분석**
```
가격 분석:
- 기본 가격대 ($0 - $100+/월)
- 가격 티어 구성
- 과금 방식 (사용자 기반? 기능 기반?)
- 엔터프라이즈 커스터마이징 옵션

예시:
┌──────────┬────────┬──────────────────────────┐
│ 서비스   │ 가격   │ 타겟 사용자             │
├──────────┼────────┼──────────────────────────┤
│ Notion   │ $0-20  │ 개인, 스타트업         │
│ Asana    │ $10-25 │ 중소기업, 팀 협업      │
│ Monday   │ $0-32  │ 중소기업, 엔터프라이즈 │
└──────────┴────────┴──────────────────────────┘
```

**4-3. 고객 포지셔닝 분석**
```
분석 항목:
- 주요 고객 프로필 (회사 규모, 산업)
- 마케팅 메시지 (강조하는 가치)
- 파트너십 전략
- 투자 현황 (성장률 지표)
```

**4-4. 온라인 평판 분석**
```
수집처:
- G2 Crowd, Capterra (사용자 리뷰, 평점)
- Product Hunt (신제품 반응)
- Twitter/Reddit (사용자 의견)
- 뉴스 기사 (시장 평판)

분석:
- 장점 (사용자가 칭찬하는 부분)
- 단점 (주요 불만사항)
- 개선 요청 (사용자가 원하는 기능)
```

#### 산출물
- 경쟁사 비교표 (기능, 가격, 포지셔닝)
- 각 경쟁사별 강점/약점 정리
- 고객 리뷰 요약

#### 체크리스트
- [ ] 각 경쟁사 제품 직접 테스트 또는 데모 영상 확인
- [ ] 기능 비교표 작성
- [ ] 가격 및 고객 분석 완료
- [ ] 사용자 리뷰 수집 (10개 이상)

---

### **Step 5: 사용자 대상 선정 및 프로필링** (2-3시간)

#### 목표
인터뷰할 사용자의 대상을 명확히 하고 프로필을 정의합니다.

#### 실행 방법

**5-1. 사용자 세분화**
```
협업 도구 사용자 세분화:

1. 직급별
   - C-level (의사결정자)
   - 팀리더 (실행자, 결정 권한 낮음)
   - 팀원 (사용자)

2. 회사 규모별
   - 스타트업 (<50명)
   - 중소기업 (50-500명)
   - 대기업 (500명+)

3. 산업별
   - IT/SaaS (기술 친화적)
   - 제조 (보수적)
   - 크리에이티브 (유연성 중시)

4. 사용 패턴별
   - 파워 유저 (일일 수십 번 사용)
   - 일반 유저 (일일 3-5회 사용)
   - 라이트 유저 (주 1-2회 사용)
```

**5-2. 타겟 사용자 정의**
```
우리의 타겟 사용자:
- 역할: 팀리더, PM
- 회사 규모: 스타트업~중소기업
- 산업: IT/SaaS
- 특징: 기술에 개방적, 효율성 중시
- 통증점: 여러 도구 사용 → 통합 솔루션 필요
- 기대사항: 빠른 셋업, 쉬운 사용법, 팀 협업
```

**5-3. 사용자 모집 계획**
```
모집 방법:
1. 개인 네트워크 (지인, LinkedIn)
2. 온라인 커뮤니티 (Reddit, Slack 커뮤니티)
3. 사용자 리서치 플랫폼 (UserTesting, Respondent)
4. 기존 고객사 (B2B의 경우)

모집 수: 5-10명 (다양성 확보)
```

#### 산출물
- 타겟 사용자 프로필 (세분화 기준 및 정의)
- 모집 계획 (방법, 일정, 보상)

#### 체크리스트
- [ ] 사용자 세분화 완료
- [ ] 타겟 사용자 프로필 정의
- [ ] 인터뷰 대상 모집 시작
- [ ] 인터뷰 보상 준비 (시간 또는 소액 보상)

---

### **Step 6: 사용자 인터뷰 및 설문 실시** (5-10시간)

#### 목표
실제 사용자로부터 니즈, 통증점, 기대사항을 파악합니다.

#### 실행 방법

**6-1. 인터뷰 준비**
```
인터뷰 질문 가이드:

I. 현재 상황 파악
  1. "현재 어떤 협업 도구를 사용하고 있나요?"
  2. "그 도구를 선택한 이유는?"
  3. "일반적인 워크플로우를 설명해주실 수 있나요?"

II. 통증점 파악
  4. "현재 도구 사용 중 가장 답답한 부분은?"
  5. "여러 도구를 함께 사용하나요? 왜?"
  6. "이로 인한 시간 낭비나 실수가 있나요?"

III. 니즈 파악
  7. "이상적인 협업 도구는 어떤 모습이어야 할까요?"
  8. "반드시 필요한 기능은?"
  9. "있으면 좋을 기능은?"

IV. 구매 의사 및 예산
  10. "새로운 도구로 전환할 의향이 있나요?"
  11. "이런 기능이 있다면 얼마까지 낼 의향이 있나요?"
  12. "의사결정 과정은 어떻게 되나요?" (B2B 경우)

V. 경쟁사 평가
  13. "노션/아사나/먼데이 중 쓰셨던 경험이?"
  14. "그 중 뭐가 가장 좋았고, 뭐가 나빴어요?"
```

**6-2. 인터뷰 실시**
```
방식:
- 1:1 심화 인터뷰 (30-60분): 5-7명
- 온라인 설문 (5-10분): 20-50명
- 포커스 그룹 (60-90분): 6-8명 (선택사항)

기록:
- 음성 녹음 (동의 후)
- 메모 (중요 포인트)
- 스크린샷 (제스처, 표정)

팁:
- 오픈 질문으로 시작 ("어떻게 생각하세요?")
- 따라 질문하기 ("더 자세히 말씀해주실 수 있나요?")
- 인터뷰어 편견 최소화 (유도 질문 피하기)
```

**6-3. 설문조사 (선택사항)**
```
설문 대상: 50-100명 (더 넓은 범위)
질문 형식:
- 객관식 (편의점 분석)
- Likert 척도 ("매우 동의 - 보통 - 동의 안함")
- 개방형 (심화 의견)

도구: Google Forms, Typeform, SurveyMonkey 등

기대치: 높은 응답률 < 깊이 있는 답변
```

#### 산출물
- 인터뷰 녹취록 또는 상세 메모
- 인터뷰 동영상 (선택)
- 설문 응답 결과 (엑셀/분석 자료)
- 사용자 인용구 모음 (임팩트 있는 문구)

#### 체크리스트
- [ ] 인터뷰 질문 가이드 작성
- [ ] 인터뷰이 5-7명 모집 완료
- [ ] 인터뷰 녹음/기록 완료
- [ ] 사용자 동의서 수집 (민감 정보 사용 시)
- [ ] 인터뷰 후 감사 메시지 전송

---

### **Step 7: 데이터 정리 및 분석** (4-6시간)

#### 목표
수집된 모든 정보를 체계적으로 분류하고 주요 인사이트를 추출합니다.

#### 실행 방법

**7-1. 인터뷰 내용 코딩 및 분류**
```
분류 카테고리:
1. 통증점 (Pain Points)
   예: "여러 도구 사용 → 컨텍스트 스위칭 비용"

2. 니즈 (Needs)
   예: "모든 기능이 한 곳에 모여있으면 좋겠어요"

3. 기대 기능 (Desired Features)
   예: "AI가 자동으로 우선순위 결정해주면?"

4. 구매 결정 요소 (Purchase Drivers)
   예: "팀원들이 쉽게 배울 수 있어야 함"

5. 경쟁사 평가 (Competitive Assessment)
   예: "노션은 자유도가 높지만 복잡해"
```

**7-2. 패턴 추출**
```
공통 테마 찾기:

인터뷰 원문:
- "여러 탭을 오갈 때마다 집중력 떨어져요" (사용자 1)
- "도구 전환할 때마다 로그인해야 해서 답답해요" (사용자 2)
- "데이터를 매번 복사/붙여넣기 해야 하는 게 싫어요" (사용자 3)

→ 공통 패턴: "Tool Fragmentation 문제"

## ❓ Q&A (자주 묻는 질문)

### Q1: 인터뷰는 몇 명을 해야 하나요?
**A**:
- 초기 탐색: 5-7명 (충분히 깊이 있는 답변)
- 정성 분석: 15-20명 (패턴 확인)
- 정량 검증: 50-100명 (통계적 유의성)

본 리서치에서는 **5-10명**이 충분합니다.

### Q2: 리서치 기간은 얼마나 걸리나요?
**A**:
- 최소: 1주 (빠른 시장 진출 필요 시)
- 표준: 2-3주 (권장)
- 심화: 4-6주 (완전한 분석 필요 시)

### Q3: 경쟁사는 정확히 몇 개를 분석해야 하나요?
**A**:
- 최소: 3개 (직접 경쟁사)
- 표준: 5개 (다양성 확보)
- 최대: 10개 이상 (시간 낭비 위험)

직접 경쟁사 3-4개 + 간접 경쟁사 1-2개 추천

### Q4: 인터뷰할 사용자를 어떻게 모나요?
**A**:
```
모집 우선순위:
1. 개인 네트워크 (가장 쉬움)
2. LinkedIn, Twitter
3. Slack 커뮤니티
4. 사용자 리서치 플랫폼 (비용 발생)

팁: 먼저 5명은 개인 네트워크에서 모으고,
    부족하면 온라인 커뮤니티 활용
```

### Q5: 인터뷰 녹음을 해도 되나요?
**A**:
**반드시 사전 동의를 받아야 합니다.**
```
동의 문구:
"이 인터뷰를 녹음해도 되실까요?
정확한 기록을 위함이고, 내용은 내부 분석용으로만 사용됩니다."
```

### Q6: 정보가 상충하면 어떻게 하나요?
**A**:
- 우선순위: 인터뷰(질적) > 설문(정량) > 온라인 평가
- 상충 시: 더 많은 데이터 수집 또는 명확하게 "이견 존재" 표시
- 의사결정: 경영진과 함께 판단

### Q7: 리서치 결과를 어떻게 공유하나요?
**A**:
```
공유 대상별:
- 경영진: Executive Summary + 10-15장 PPT
- 제품팀: 전체 리서치 보고서 (상세)
- 마케팅팀: 시장/고객 분석 부분
- 개발팀: 기대 기능 부분

공유 방식: 1) 문서 공유 2) 발표 미팅
```

### Q8: 리서치 비용이 얼마나 드나요?
**A**:
```
비용 구성:
- 내부 시간: 100-150시간 ($5K-15K 상당)
- 도구/구독: $500-2K (선택사항)
- 사용자 보상: $500-2K (인터뷰 참여비)
- 외부 리서처: $10K-50K (용역 시)

초기 스타트업: 내부 인력으로 $0-1K (도구만)
성숙 기업: 전문가 용역 $20K-50K
