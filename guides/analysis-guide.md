# 📊 분석 에이전트 상세 가이드

> 리서치 데이터를 심층 분석하여 인사이트 도출
> 경쟁사 포지셔닝, 사용자 패턴, 차별화 포인트 발굴

---

## 📋 개요

### 에이전트 목표
**리서치 에이전트가 수집한 정보를 체계적으로 분석하여, 기획의 방향을 결정할 수 있는 인사이트를 제공합니다.**

### 언제 사용하는가?

✅ **필수**:
- 리서치 보고서가 완성되었을 때
- 시장 기회를 찾아야 할 때
- 경쟁사와의 차별화 포인트를 정해야 할 때
- 사용자 니즈의 우선순위를 파악해야 할 때

❌ **불필요**:
- 리서치가 아직 진행 중일 때
- 이미 분석 완료된 데이터가 있을 때 → 직접 구조화 에이전트 사용

### 기대되는 결과

| 산출물 | 설명 | 용도 |
|--------|------|------|
| 시장 분석 보고서 | 패턴, 인사이트, 기회 | 전략 수립 기초 |
| 경쟁 포지셔닝 맵 | 경쟁사 위치 시각화 | 차별화 전략 |
| 사용자 니즈 우선순위 | 기능 우선순위 매트릭스 | 아이데이션 입력 |
| 차별화 포인트 정의 | 우리의 차별화 전략 | 기획 방향 |
| SWOT 분석 | 강점/약점/기회/위협 | 리스크 관리 |

---

## 🎯 단계별 프로세스 (8 Steps)

### **Step 1: 분석 목표 및 프레임워크 선정** (1-2시간)

#### 목표
어떤 관점으로 분석할지, 어떤 프레임워크를 사용할지 결정합니다.

#### 실행 방법

**1-1. 분석 질문 정의**
```
예시:
Q1: "우리가 어떤 시장 세그먼트를 공략할 것인가?"
Q2: "경쟁사 대비 우리의 차별화 요소는 무엇인가?"
Q3: "해결해야 할 사용자의 Top 3 통증점은?"
Q4: "우리만 할 수 있는 것은 무엇인가?"
```

**1-2. 분석 프레임워크 선정**
```
추천 프레임워크:

1. Porter의 Five Forces (경쟁 강도 분석)
   ├─ 신규 진입자 위협
   ├─ 대체제 위협
   ├─ 공급자 협상력
   ├─ 구매자 협상력
   └─ 기존 경쟁사 간 경쟁

2. SWOT 분석 (강점/약점/기회/위협)
   ├─ Strengths (내 강점)
   ├─ Weaknesses (내 약점)
   ├─ Opportunities (시장 기회)
   └─ Threats (시장 위협)

3. Market Segmentation (시장 세분화)
   ├─ 지리적 세분화 (지역별)
   ├─ 인구통계학 (회사 규모, 산업)
   ├─ 심리적 (가치관, 라이프스타일)
   └─ 행동학적 (사용 패턴)

4. Positioning Map (포지셔닝 맵)
   ├─ X축: 가격 (저←→고)
   ├─ Y축: 기능 (심플←→복잡)
   └─ 거품 크기: 시장 점유율

5. Gap Analysis (기회 분석)
   ├─ 시장이 원하는 것
   ├─ 경쟁사가 제공하는 것
   ├─ 격차 (Gap)
   └─ 우리의 기회
```

**1-3. 분석 범위 결정**
```
범위 설정:
- 시간: 1년? 3년? 5년?
- 지역: 한국? 글로벌?
- 고객: SMB? 엔터프라이즈? 모두?
```

#### 체크리스트
- [ ] 분석 질문 3-5개 정의
- [ ] 프레임워크 1-2개 선택
- [ ] 분석 범위 명확히
- [ ] 팀 동의 확보

### **Step 2: 리서치 데이터 정제 및 구조화** (2-3시간)

#### 목표
산발적인 리서치 데이터를 정리하여 분석 가능하게 준비합니다.

#### 실행 방법

**2-1. 데이터 검수**
```
검수 항목:
- 오류 데이터 식별 (이상치)
- 중복 제거
- 출처 확인 (신뢰도 평가)
- 최신성 확인 (오래된 데이터 주의)

예시:
❌ 경쟁사 A 시장점유율: 150% (오류, 제거)
✅ 경쟁사 A 시장점유율: 15% (정확)
```

**2-2. 데이터 분류**
```
분류 카테고리:

1. 시장 데이터
   - 시장 규모, 성장률
   - 산업 트렌드
   - 규제 환경

2. 경쟁사 데이터
   - 기능 비교
   - 가격 모델
   - 고객 리뷰

3. 사용자 데이터
   - 니즈 (목표)
   - 통증점 (문제)
   - 행동 패턴
   - 구매 결정 요소

4. 환경 데이터
   - 기술 트렌드
   - 경제 상황
   - 사회문화 변화
```

**2-3. 구조화 (엑셀/시트 정리)**
```
예시 구조:

┌──────────┬────────┬────────┬──────┬────────┐
│ 데이터   │ 출처   │ 날짜   │ 신뢰 │ 비고   │
├──────────┼────────┼────────┼──────┼────────┤
│ 시장규모 │ 보고서 │ 2024   │ 높음 │ 예상치 │
│ 경쟁사A  │ G2     │ 2024.1 │ 중간 │ 유저평 │
│ 사용자1  │ 인터뷰 │ 2024.1 │ 높음 │ 원본   │
└──────────┴────────┴────────┴──────┴────────┘
```

#### 산출물
- 정제된 데이터셋 (엑셀)
- 데이터 품질 보고서

#### 체크리스트
- [ ] 이상치 제거 완료
- [ ] 데이터 분류 완료
- [ ] 출처 기록 완료
- [ ] 신뢰도 평가 완료

### **Step 3: 경쟁사 기능 비교 분석** (2-3시간)

#### 목표
경쟁사의 기능을 체계적으로 비교하여 차별화 포인트를 발굴합니다.

#### 실행 방법

**3-1. 기능 매트릭스 작성**
```
기능 카테고리 정의:

1. 핵심 기능 (Core Features)
   - 프로젝트 관리
   - 협업 기능
   - 리포팅

2. 차별화 기능 (Differentiators)
   - AI 자동화
   - 고급 분석
   - 통합 능력

3. 부가 기능 (Nice-to-have)
   - 테마 커스터마이징
   - 모바일 앱
   - 오프라인 모드

매트릭스:
┌──────────┬───────┬───────┬───────┐
│ 기능     │ 우리  │ 경쟁A │ 경쟁B │
├──────────┼───────┼───────┼───────┤
│ 칸반보드 │ ✓     │ ✓     │ ✓     │
│ AI 자동화│ ✓     │ ✗     │ △     │
│ 가격     │ $15   │ $20   │ $25   │
└──────────┴───────┴───────┴───────┘
```

**3-2. 기능 심화 분석**
```
각 기능별 분석:

기능: AI 자동화
├─ 경쟁사 A
│  ├─ 보유: 미보유 (✗)
│  ├─ 이유: 개발 중 (예상)
│  └─ 영향: 중간 (고객 요청 많음)
│
└─ 경쟁사 B
   ├─ 보유: 베타 (△)
   ├─ 이유: 최근 출시
   └─ 영향: 높음 (마케팅 강조)

→ 해석: AI 자동화는 우리의 차별화 포인트!
```

**3-3. 기능 우선순위 평가**
```
우선순위 기준:

1. 필수도 (Must Have): 없으면 안 됨
2. 중요도 (Should Have): 경쟁에서 필요
3. 가치도 (Nice to Have): 있으면 좋음
4. 개발 난이도: 쉬움/중간/어려움

우선순위 매트릭스:
   ▲
   │ 어려움 │  ●●
   │        │ ●   ●
   │ 쉬움   │    ●
   └───────────────────→ 가치도

   → 오른쪽 아래가 가장 먼저 개발할 기능
```

#### 산출물
- 기능 비교 매트릭스
- 각 기능별 심화 분석
- 우선순위 평가표

#### 체크리스트
- [ ] 기능 카테고리 정의
- [ ] 비교 매트릭스 작성
- [ ] 각 기능 심화 분석
- [ ] 우선순위 결정

### **Step 4: 사용자 니즈 패턴 도출** (3-4시간)

#### 목표
사용자 인터뷰/설문 데이터로부터 공통 패턴을 찾아 카테고리화합니다.

#### 실행 방법

**4-1. 니즈 코딩 및 분류**
```
니즈 분류:

1. 기능 니즈 (Functional Need)
   "자동으로 우선순위를 정해줬으면"
   → 기능: AI 기반 우선순위 자동화

2. 감정 니즈 (Emotional Need)
   "너무 복잡해서 스트레스 받아"
   → 방향: 심플한 UX 설계

3. 사회 니즈 (Social Need)
   "팀이 함께 쓸 수 있어야 해"
   → 기능: 협업/권한 관리

4. 가치 니즈 (Value Need)
   "돈의 가치를 느껴야 해"
   → 방향: 명확한 ROI 제시

5. 편의 니즈 (Convenience Need)
   "모바일에서도 빠르게 접근 가능해야 해"
   → 기능: 우수한 모바일 UX
```

**4-2. 니즈 우선순위화**
```
우선순위 결정 방법:

1. 빈도 (Frequency)
   - 얼마나 많은 사용자가 언급했는가?
   - 예: 7명 중 6명 = 86%

2. 심각도 (Severity)
   - 그 문제가 얼마나 심각한가?
   - 예: "못 써먹을 정도" vs "좀 불편"

3. 만족도 갭 (Satisfaction Gap)
   - 현재와 기대의 차이는?
   - 예: 기대 8점 vs 현재 3점 = 5점 갭

우선순위 점수:
우선순위 = 빈도(%) × 심각도(1-5) × 갭(1-5) / 100

예시:
니즈A: 86% × 5 × 5 = 21.5점 (최우선)
니즈B: 71% × 3 × 4 = 8.5점 (2순위)
니즈C: 43% × 2 × 3 = 2.6점 (3순위)
```

**4-3. 니즈 맵핑**
```
사용자 세그먼트별 니즈:

개인 사용자 (Personal):
├─ 단순성: 높음
├─ 가격: 민감
└─ 모바일: 필수

팀리더 (Team Leader):
├─ 협업: 필수
├─ 리포팅: 필수
└─ 가격: 중간

엔터프라이즈 (Enterprise):
├─ 보안: 필수
├─ 커스터마이징: 필수
└─ 가격: 낮음 (수익 대비)

→ 각 세그먼트별 전략 수립!
```

#### 산출물
- 니즈 분류표 (코딩된 데이터)
- 니즈 우선순위 매트릭스
- 세그먼트별 니즈 맵

#### 체크리스트
- [ ] 니즈 코딩 및 분류 완료
- [ ] 우선순위 스코어 계산
- [ ] 세그먼트별 니즈 맵핑
- [ ] Top 5 니즈 확인

### **Step 5: 기회요인 및 위협요인 식별** (2-3시간)

#### 목표
시장 분석을 통해 우리가 공략할 기회와 주의할 위협을 명확히 합니다.

#### 실행 방법

**5-1. 시장 기회 발굴**
```
기회 발굴 방법:

1. Gap 분석: 시장이 원하지만 아무도 제공 안 하는 것
   예: "AI 자동화 기능" → 우리가 먼저 제공 가능

2. Trend 기회: 새로운 트렌드가 만드는 기회
   예: "리모트워크 확산" → 협업 도구 수요 증가

3. 경쟁사 약점: 경쟁사의 약점에서 오는 기회
   예: "노션은 가격이 싼데 기능이 복잡" → 우리는 심플하게

4. 신규 시장: 아직 개척되지 않은 시장
   예: "아프리카 시장 미개척" → 지역화 전략

기회 평가:
┌────────────┬────────┬────┐
│ 기회       │ 시장성 │ 우리 │
├────────────┼────────┼────┤
│ AI 자동화  │ 높음   │ 높음 │ ← 최우선
│ 리모트 UX  │ 중간   │ 높음 │
│ 가격 경쟁  │ 낮음   │ 중간 │
└────────────┴────────┴────┘
```

**5-2. 위협요인 식별**
```
위협 유형:

1. 경쟁사 위협
   - 기존 강자의 기능 추가
   - 신규 진입자의 혁신
   - 예: "Microsoft가 Teams에 AI 추가" → 우리 기능 우월성 하락

2. 시장 변화 위협
   - 고객 선호 변화
   - 규제 변화
   - 예: "GDPR 강화" → 데이터 처리 비용 증가

3. 기술 위협
   - 새로운 기술 출현
   - 기술 플랫폼 변화
   - 예: "AI 모델 고도화" → 누구나 자동화 기능 만들 수 있음

4. 경제 위협
   - 경기 침체
   - 고객 예산 감소
   - 예: "금리 인상" → B2B 투자 감소

위협 평가:
┌────────────┬────┬──────┐
│ 위협       │ 발생 │ 영향 │
├────────────┼────┼──────┤
│ MS Teams   │ 높음 │ 높음 │ ← 모니터링 필수
│ 경기 침체  │ 중간 │ 높음 │
│ 신규 진입  │ 높음 │ 중간 │
└────────────┴────┴──────┘
```

**5-3. 기회-위협 대응 전략**
```
전략 매트릭스:

기회 × 우리 강점 = 적극 추진
예: AI 자동화 기능 우리가 잘함 → 최우선 개발

기회 × 우리 약점 = 파트너십 또는 투자
예: 엔터프라이즈 보안 우리 약함 → 보안 회사 제휴

위협 × 우리 약점 = 즉시 대응
예: MS Teams 기능 추가 + 우리 기능 약함 → 긴급 개발

위협 × 우리 강점 = 모니터링
예: 신규 진입 + 우리 먼저 출시 → 시장 우위 유지
```

#### 산출물
- 시장 기회 분석
- 위협요인 식별표
- 기회-위협 대응 전략

#### 체크리스트
- [ ] 시장 기회 3-5개 발굴
- [ ] 위협요인 3-5개 식별
- [ ] 각 기회/위협의 영향도 평가
- [ ] 대응 전략 수립

### **Step 6: 차별화 포인트 정의** (2-3시간)

#### 목표
경쟁사와 구별되는 우리만의 강점을 명확히 정의합니다.

#### 실행 방법

**6-1. 차별화 차원 분석**
```
차별화 차원:

1. 기능 차별화 (Feature)
   "우리만 AI 자동화가 있다"
   장점: 명확, 복사 가능
   단점: 경쟁사가 따라할 수 있음

2. 가격 차별화 (Price)
   "가장 싼 가격이다"
   장점: 명확, 초기 고객 확보
   단점: 이윤 낮음, 가격 전쟁

3. UX 차별화 (User Experience)
   "가장 쉽고 빠르다"
   장점: 사용자 만족도 높음
   단점: 객관화 어려움

4. 서비스 차별화 (Service)
   "가장 좋은 지원팀"
   장점: 충성도 높음
   단점: 스케일 어려움

5. 브랜드 차별화 (Brand)
   "우리는 신뢰할 수 있는 기업"
   장점: 프리미엄 가격 가능
   단점: 구축 시간 오래 걸림

최선: 여러 차원 결합
예: 좋은 기능 + 저가격 + 우수 UX + 좋은 서비스
```

**6-2. 차별화 포인트 검증**
```
차별화 포인트 평가 기준:

1. 고객이 원하는가? (Customer Demand)
   - 고객이 이것을 중요하게 생각하는가?
   - 예: "AI 자동화" → 사용자 86% 원함 (✓)

2. 우리가 잘하는가? (Our Capability)
   - 우리에게 경쟁사보다 우위가 있는가?
   - 예: "AI 자동화" → 우리 AI 팀 강함 (✓)

3. 경쟁사와 다른가? (Differentiation)
   - 경쟁사가 따라하기 어려운가?
   - 예: "AI 자동화" → 누구나 따라 가능 (△)

4. 지속 가능한가? (Sustainability)
   - 장기적으로 유지할 수 있는가?
   - 예: "저가격" → 원가 하락으로만 가능 (△)

평가 표:
┌──────────────┬───┬───┬───┬───┬─────┐
│ 차별화 포인트│ 1 │ 2 │ 3 │ 4 │ 점수 │
├──────────────┼───┼───┼───┼───┼─────┤
│ AI 자동화    │ ✓ │ ✓ │ △ │ △ │ 2.5 │
│ 심플한 UX    │ ✓ │ ✓ │ ✓ │ ✓ │ 4.0 │ ← 최고
│ 저가격       │ ✓ │ △ │ ✗ │ △ │ 1.5 │
└──────────────┴───┴───┴───┴───┴─────┘
```

**6-3. 최종 차별화 전략 수립**
```
차별화 포지셔닝:

주요 차별화 (Primary): 1개
"심플하고 강력한 협업 플랫폼"
- 모든 마케팅이 이것에 집중
- 모든 기능 개발이 이것에 부합

보조 차별화 (Secondary): 2-3개
"AI 기반 우선순위 자동화"
"최고 수준의 사용성"

방어선 (Defense): 1-2개
"중간대 가격대"
"빠른 고객 지원"

예시 포지셔닝 맵:
    ▲ 기능 완성도
    │         ●경쟁사A (복잡, 기능 완성)
    │    ●우리 (심플, 강력)
    │
    │              ●경쟁사B (간단, 기능 부족)
    └──────────────────────→ 가격
          저          고
```

#### 산출물

- 차별화 포인트 평가표
- 최종 포지셔닝 정의

#### 체크리스트

### **Step 7: SWOT 분석 작성** (1-2시간)

#### 목표
지금까지의 분석을 SWOT으로 통합하여 전체 상황을 파악합니다.

#### 실행 방법

**7-1. SWOT 각 항목 작성**
```
Strengths (우리의 강점):
- AI/ML 기술력 (우수)
- 사용자 중심 UX 철학
- 빠른 반응 속도 (스타트업 문화)
- 유연한 커스터마이징
+ 종합: 혁신적이고 고객 친화적

Weaknesses (우리의 약점):
- 엔터프라이즈 고객 경험 부족
- 마케팅/브랜드 인지도 낮음
- 24/7 지원팀 부족
- 초기 자본금 한정
- 종합: 스케일 능력 부족

Opportunities (시장 기회):
- AI 트렌드 확산 (수요 증가)
- 리모트워크 일반화 (협업 도구 수요)
- 경쟁사의 높은 가격대 (가격 리더십 기회)
- 아직 미개척 시장 (해외 진출)
+ 종합: 고성장 시장 기회

Threats (시장 위협):
- 기존 강자(Microsoft)의 기능 추가
- 신규 진입자의 가격 경쟁
- 고객 예산 감소 (경기 침체)
- 기술 표준화 (누구나 AI 가능)
+ 종합: 경쟁 심화 및 마진 하락
```

**7-2. SWOT 교차 분석**
```
교차 분석: 전략 도출

1. S × O (추진 전략)
   "우리의 강점으로 시장 기회를 공략하자"
   강점: 우수한 UX
   기회: 리모트워크 확산
   → 전략: 리모트 팀을 위한 최고 UX 제공

2. W × O (개선 전략)
   "시장 기회로 우리의 약점을 보완하자"
   약점: 엔터프라이즈 경험 부족
   기회: 높은 시장 성장
   → 전략: 고성장 SMB 먼저 공략 후 업그레이드

3. S × T (방어 전략)
   "우리의 강점으로 시장 위협을 대비하자"
   강점: 빠른 반응 속도
   위협: 경쟁사 기능 추가
   → 전략: 지속적 혁신으로 리더십 유지

4. W × T (회피 전략)
   "우리의 약점이 되는 위협을 피하자"
   약점: 마케팅 약함
   위협: 가격 경쟁
   → 전략: 가격 전쟁 피하고 가치 강조
```

**7-3. SWOT 시각화 및 정리**
```
SWOT 매트릭스:

┌─────────────┬──────────────────┬──────────────────┐
│             │ STRENGTH (강점)  │ WEAKNESS (약점)  │
├─────────────┼──────────────────┼──────────────────┤
│ OPPORTUNITY │ SO 전략           │ WO 전략          │
│ (기회)      │ • 혁신으로 성장   │ • SMB 먼저 공략  │
│             │ • AI 리더십       │ • 파트너십       │
├─────────────┼──────────────────┼──────────────────┤
│ THREAT      │ ST 전략           │ WT 전략          │
│ (위협)      │ • 계속 혁신       │ • 가치 강조      │
│             │ • 선점 경쟁       │ • 니치 시장      │
└─────────────┴──────────────────┴──────────────────┘
```

#### 산출물
- SWOT 분석 정리표
- SWOT 교차 분석 (전략)
- 1장 요약 (임팩트 있는 정보)

#### 체크리스트

## ❓ Q&A (자주 묻는 질문)

### Q1: 경쟁사는 정확히 몇 개를 심화 분석해야 하나요?
**A**: 3-5개 추천
- 직접 경쟁사: 3개 (필수)
- 간접 경쟁사: 1-2개 (참고용)

너무 많으면 분석 피로도 증가

### Q2: SWOT는 한 번만 하면 되나요?
**A**: 아니오, 주기적 업데이트 필요
- 초기: 기획 단계 (지금)
- 정기: 분기별 (변화 모니터링)
- 혹은: 경쟁 상황 변화 시

### Q3: 데이터가 상충하면 어떻게 하나요?
**A**:
1. 데이터 신뢰도 평가
2. 더 최근 데이터 우선
3. "이견 존재" 명확히 표시
4. 향후 조사 항목으로 추가

### Q4: 분석에 얼마나 시간이 걸리나요?
**A**:
- 최소: 1주 (리서치 완료 후)
- 표준: 2주
- 심화: 3-4주

### Q5: 분석 결과가 리서치와 다르면?
**A**: 정상입니다!
- 리서치: 수집 (What)
- 분석: 해석 (Why/So What)

다르다면 더 깊은 인사이트!
